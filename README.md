#  AdGuard AI — Multi-Agent Real-Time Bidding Fraud Detection on Microsoft Foundry


> **Microsoft IQ Layer:** Foundry IQ — Azure AI Search RAG



##  What This Project Does

The global programmatic advertising industry loses **$100 billion annually** to ad fraud.
Advertisers pay for audiences that never exist — invisible stacked ads, robot click farms,
and deliberately subtle impression fraud drain budgets silently while traditional systems
catch it too late with no legal justification.

**AdGuard AI** is an autonomous 5-agent system built on Microsoft Foundry that monitors
real-time bidding traffic, detects fraud the moment it happens, enforces automated blocks,
and delivers a legally-grounded audit report — all without human intervention, in under
30 seconds.



##  Agent Responsibilities

Each agent has a single clearly defined responsibility. No agent overlaps with another.

| Agent | Name | Responsibility | Tool | Cost |
|---|---|---|---|---|
| Agent 1 | Traffic Monitor | Scans 200 RTB records using format-aware CTR thresholds. Flags suspicious records. | Pure Python — no API | Free |
| Agent 2 | Foundry IQ | Retrieves real IAB/MRC compliance law from Azure AI Search to ground AI verdicts | Azure AI Search | Free F0 |
| Agent 3 | Classifier | Reads flagged records and compliance law. Identifies fraud type, maps exact breach code, calculates damage | Microsoft Foundry gpt-4.1-mini | ~$0.001 |
| Agent 4 | Blocker | Issues automated publisher blacklist, IP firewall block, adset disable logs | Microsoft Foundry gpt-4.1-mini | shared call |
| Agent 5 | Reporter | Writes executive audit report with total damage, budget saved, and recommendations | Microsoft Foundry gpt-4.1-mini | shared call |

> Agents 3, 4, and 5 run in a **single token-optimised Microsoft Foundry call** — reducing latency and cost while each agent maintains a distinct persona, mandate, and output format.

##  Tools and APIs Used

| Tool | Agent | Purpose | Connection |
|---|---|---|---|
| Azure AI Search | Agent 2 | Foundry IQ knowledge retrieval — fetches IAB/MRC compliance docs | azure-search-documents SDK |
| Microsoft Foundry gpt-4.1-mini | Agents 3/4/5 | Fraud classification, enforcement, reporting | AzureOpenAI SDK |
| Streamlit | UI layer | Dashboard rendering, chart display, pipeline trigger | Streamlit Cloud |
| Python pandas | Agent 1 | Data processing and anomaly detection | Local library |
| python-dotenv | Config | Secure key management | Local library |

>Azure AI Search is connected as a Foundry-compatible knowledge tool following Model Context Protocol principles for grounded, permission-aware retrieval. Agent 2 queries the index and injects retrieved context directly into the Agent 3/4/5 prompt — the RAG pattern that Foundry IQ is designed to support.

##  Three Fraud Types Detected

| Fraud Type | Publisher | Breach Code | Detection Signal |
|---|---|---|---|
| Ad Stacking | pub_vortex_media | IAB-AS-009 | imp_bid_ratio above 5x on display_banner |
| Botnet Click Farm | pub_click_nexus | IAB-BF-100 | CTR 35-45% on rewarded_video from static IP |
| Low-and-Slow Fraud | pub_shadow_network | IAB-LS-200 | imp_bid_ratio 1.5x-1.9x sustained across 40 records |



###  Foundry IQ — Azure AI Search (Active)
Agent 2 implements Foundry IQ — agentic knowledge retrieval grounded on 7 IAB/MRC
compliance documents stored in Azure AI Search index fraud-playbook. Every Agent 3
verdict is cited against real industry law — eliminating hallucination and making
verdicts legally defensible.


## Responsible AI Controls

| Control | Implementation |
|---|---|
| Hallucination prevention | Agent 2 retrieves real IAB law before Agent 3 writes verdicts — AI never recalls breach codes from memory |
| Synthetic data only | All data generated programmatically — no real user, advertiser, or publisher data |
| No silent fallback | Agent 2 fails loudly if Azure Search returns no results |
| Hard stop on missing config | App stops immediately if any Azure credential is missing |
| Token safety cap | Max 30 anomaly records sent to Foundry — prevents context overflow |
| Output validation | Agent 3 verdicts checked against known publisher IDs — hallucinated publishers flagged |
